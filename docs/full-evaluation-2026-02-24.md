# KidsNote 全方位評価 — Claude × Codex 議論結果

**日付**: 2026-02-24
**議論方式**: Claude Code × Codex MCP による構造化議論（アンカリング防止プロトコル）

---

## 1. 総合評価

KidsNoteは現時点では実運用基盤が未整備のため**全シナリオでNo-Go**だが、AI音声記録を核にした価値は明確であり、**安全/法令適合 → 現場効果実証 → TCO妥当化**の順でゲートを通過できれば「導入したい」に到達可能。

評価軸は導入側視点で以下に固定:

**安全/法令適合 > 現場効果(実測) > TCO > 戦略価値**

## 2. 現状の強み

- **AI音声入力 → 7intent自動分類**の記録導線が速い（競合にない差別化）
- 成長記録の情報粒度が高い（5領域レベル追跡・要録マーク・公開範囲）
- ヒヤリハットの重症度・エスカレーション設計がしっかりしている
- 申し送りの既読/至急管理で園内連携に使える
- 出席簿・シフト・出勤簿の運用画面が一通り揃っている
- 園ルールCRUD + AI QAで規程参照の即時性が高い
- **学校基本調査準拠の設定と自動集計**（行政対応の強み）

1. **Gate A完全クリア** — 法令・安全・データ保全の運用監査通過

## 7. UX改善提案

| 改善項目 | 内容 |
|---|---|
| **自動保存モデル** | confidence≥0.9 かつ安全語なし → 自動保存+取消トースト（Gmail送信取消モデル） |

| **期待落差解消** | 未実装ページは非表示または「準備中」を明示 |
| **初回体験** | 3分チュートリアル + サンプル入力ガイド |
| **日次クロージング** | 「未処理申し送り」「ヒヤリハット0件確認」ダイアログ |

## 9. 差別化戦略

### 核心的な差別化

「音声入力単体」ではなく、**「記録→5領域評価→要録ドラフト自動生成」の縦バリューチェーン**を差別化の核にする。

### 戦略方針

- 競合との比較軸を機能数ではなく**「90日で削減できた分/人/日」**で示す
- テナント分離前提で個人情報を学習に使わない運用で信頼性を担保
- Hybrid戦略（既存SaaS + AI記録アドオン）も並行検討し導入障壁を下げる

### 差別化の持続条件

- 5領域評価を日常記録に埋め込み、年度末要録へ自然接続する仕組みの構築
- 音声入力単体は競合に6-12ヶ月で追随される可能性があるため、バリューチェーンの深さで勝負

6. UX改善の最小実装対象決定（自動保存・警告UI・サイドバー簡素化）
7. 1クラスPoC実施計画（対象者・計測方法・週次レビュー体制）確定

| AI分類精度 | ≥80%（95%CI下限） | 手動レビューサンプリング |

| 個人情報漏洩 | 1件でも | 即時停止。セキュリティ監査 |

## 結論

KidsNoteのAI記録コアは「幼稚園の記録業務を根本的に変える」ポテンシャルを持っている。しかし現時点で「導入したい」と管理者が判断するには、**Gate A（安全/法令基盤）の解消が絶対条件**。Phase 0に集中し、まず「安心して使える土台」を作ることが最優先。

その上で、「記録→5領域評価→要録」の縦バリューチェーンを段階的に構築することで、競合にない持続的な差別化を実現できる。
